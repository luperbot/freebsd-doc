6.4. Meta caratteri

La shell e alcuni programmi di manipolazione testo permettono i meta-caratteri, chiamati anche wild card, i quali vengono rimpiazzati dai corrispondenti schemi (pattern). Per i nomi di file questi meta-caratteri e i loro significati sono:

?indica un singolo carattere alla posizione indicata
*indica una stringa di zero o più caratteri
[abc...]indica un carattere tra quelli racchiusi
[a-e]indica un carattere tra quelli nel range a, b, c, d, e
[!def]indica un carattere tra quelli non inclusi in parentesi, solamente sh(1)
{abc,bcd,cde}indica un set di caratteri tra quelli inclusi in parentesi, separati da una virgola (,) (niente spazi), solamente csh(1)
~indica la directory home dell'utente corrente, solamente csh(1)
~userindica la directory home dell'utente specificato, solamente csh(1)
